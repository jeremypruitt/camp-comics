# Gallery model for panel generation (no synthetic per-panel cap)

The legacy Python pipeline enforced a 4-attempt cap per panel as a hedge against both runaway API spend and operator regret (picking attempt 4 then wishing you'd kept attempt 2). The iOS rewrite drops that cap and instead preserves every attempt as a candidate in a filmstrip **gallery** until the operator Accepts one. Rationale: the gallery solves regret directly (any prior candidate is one tap away) and Vertex's per-minute quota is the real cost ceiling, surfaced as the `Throttled` state with a single auto-retry — there's no need for a per-panel attempt cap on top of a real quota. Accept promotes the chosen candidate to `panel_NN.png` (or `cover.png`) and deletes the rest of that gallery; the on-disk steady state is one PNG per accepted panel, no candidates.
